У тебя есть диалог между несколькими участниками. Он представлен в следующем формате:

[speaker_0]: текст_спикера_0  
[speaker_1]: текст_спикера_1  
...


Твоя задача:
1. Найди задания, упомянутые в разговоре, и назначь их соответствующим людям.
2. Определи крайние сроки (дедлайны) для найденных тобой задач в формате "час":"минута" "день":"месяц":"год".
3. Назначь задания только в том случае, если это логично по контексту. Если в разговоре не говорится, что человек должен выполнить задачу, не придумывай ее.
4. Оформи задания на том же языке, на котором ведется диалог.
5. Выведи результат в формате JSON. Каждое задание должно быть представлено как объект с полями "task" (задача) и "time" (время на выполнение). Не добавляй никаких дополнительных символов, кроме самого JSON.

Чтобы тебе было проще ориентироваться в датах дедлайнов, вот текущая дата (это не дата, когда был записан диалог, а текущая дата - она с большей вероятностью близка к дате диалога, но не обзятельно. 
Поэтому тебе нужно будет постараться с помощью всей имеющейся у тебя информации о времени правильно определить дедлайны задач):
[CURRENT_DATE]

Пример вывода:

{
  "Саша": [
    {
      "task": "Написать документ о...",
      "time": "12:30 10.05.2025"
    },
    {
      "task": "Сделать созвон с...",
      "time": "10:25 15.05.2025"
    }
  ],
  "Данил": [
    {
      "task": "Реализовать...",
      "time": "15:10 17.08.2025"
    }
  ],
  "Женя": [
    {
      "task": "...",
      "time": "..."
    },
    {
      "task": "...",
      "time": "..."
    }
  ]
}


ОЧЕНЬ ВАЖНО:
1. Не назначайте задания случайным образом — давайте их только тем, кому они действительно поручены или могут быть логично даны.
2. Каждое задание должно иметь разумную оценку времени на выполнение.
3. Не нужно оборачивать вывод в кодовые блоки.
4. Если в разговоре не упомяналя крайний срок для задачи, тогда напиши вместо даты прочерк: "-".
5. Если в разговоре не указанно конкретное время крайного срока, но сама дата указанна, указывай время как "00:00:00".
6. Если в разговоре указанно только примерное время дня (например утро, день, вечер), тогда указывай время соответственно этим примерным оценкам времени (например, для утра - 9:00, для дня - 12:00 и т.д)
7. Ещё раз обращаю твоё внимание на то, что тебе нужно именно расчитать даты задач - они весьма не очевидны. Наппример, если в разговоре сначала говорится одна дата, а затем говорится день недели, 
ты должен сделать логичный вывод, что указанный день недели точно находится рядом с этой датой.
8. Если указанно только время, но не указанна дата, то указывай сегодняшнюю дату.
9. Если ну прям вообще никак не удаётся назначить задачи, тогда напиши кодовое слово [CANT_HANDLE].

Дополнительно:
1. Диалог, который ты получишь, был извлечён из аудиофайла путём транскрибации, и, как следствие, в тексте могут присутствовать ошибки из-за того, что модель транскрибации
плохо расслышала текст. Поэтому, когда ты будет рассписывать задачи, постарайся написать их с исправленными ошибками транскрибации. Вот пример:
```
Транскрибированный текст:
  Уже темнеет, нам пора *помыть* домой

Тест с исправленными ошибками транскрибации:
  Уже темнеет, нам пора *пойти* домой
```
